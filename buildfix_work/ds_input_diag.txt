## 问题背景

线上诊断接口 /api/admin/diagnostics 返回 database 类别 1/9 fail（8/9 表显示 0 条），但 Supabase 里数据是有的（例如 job_descriptions 4756 条）。

根因已定位：execSql 走 Supabase RPC `exec_sql`，但该 RPC 函数 search_path 未设 public，所以裸表名 `SELECT COUNT(*) FROM 表名` 查不到表。curl 验证：`SELECT count(*) FROM public.job_descriptions` 能返回 4756。

- 不动数据库
- 不引入新 import
- 不改动 tables 数组以外的核心逻辑

## src/app/api/admin/diagnostics/route.ts 中 tables 数组与 execSql 调用片段

```ts
// 数据库状态检查
async function testDatabase(): Promise<any[]> {
  const tables = [
    { name: 'job_descriptions', min: 1000, display: '岗位数据' },
    { name: 'skill_taxonomy', min: 100, display: '技能分类' },
    { name: 'skill_relations', min: 100, display: '技能关系' },
    { name: 'career_paths', min: 10, display: '职业路径' },
    { name: 'interview_questions', min: 10, display: '面试题库' },
    { name: 'skill_assessments', min: 10, display: '技能测评' },
    { name: 'learning_resources', min: 10, display: '学习资源' },
    { name: 'articles', min: 1, display: '文章内容' },
    { name: 'user_profiles', min: 0, display: '用户数据' },
  ];

  return Promise.all(
    tables.map(async (table) => {
      try {
        const result = await execSql(
          `SELECT COUNT(*)::int as count FROM ${table.name}`
        ) as Array<Record<string, unknown>>;
        const count = (result?.[0]?.count as number) || 0;
        const passed = count >= table.min;
        return {
          name: table.display,
          table: table.name,
          count,
          min: table.min,
          result: passed ? 'pass' : 'fail',
          detail: `${count}/${table.min}`
        };
      } catch (e: any) {
        return {
          name: table.display,
          table: table.name,
          count: 0,
          min: table.min,
          result: 'fail',
          detail: e.message
        };
      }
    })
  );
}
```
