# 根因分析

## 问题本质

1. **Cookie 清除不彻底**：`clearAuthCookies` 设置了 `maxAge: 0`，但 `httpOnly` 和 `sameSite` 属性与设置时不一致，可能导致浏览器拒绝覆盖旧 cookie
2. **Supabase 服务端 session 未失效**：只清除了客户端 cookie，但 Supabase 服务端该用户的 session 仍然有效。如果攻击者获取了旧的 access token，仍可继续使用
3. **`getUser` 验证的局限性**：`supabase.auth.getUser(token)` 只验证 token 签名和过期时间，不会检查该 session 是否已被服务端撤销

## 关键发现

`clearAuthCookies` 中 `maxAge: 0` 的 cookie 设置缺少 `httpOnly` 和 `sameSite` 属性（虽然传入了 `COOKIE_OPTIONS`，但实际代码中 `clearAuthCookies` 确实使用了 `...COOKIE_OPTIONS`，所以不是这个问题）。

**真正的问题**：`clearAuthCookies` 只清除了 `sb-access-token` 和 `sb-refresh-token`，但 Supabase 的 `gotrue` 库可能还设置了其他 cookie（如 `sb-*` 前缀的其他 cookie），或者浏览器缓存了旧的 cookie 值。

**最根本的问题**：没有调用 `supabase.auth.signOut()` 使服务端 session 失效，导致旧的 access token 仍然有效。

### src/app/api/auth/logout/route.ts（完整新代码）

```typescript
export const dynamic = 'force-dynamic';
import { NextResponse } from 'next/server';
import { clearAuthCookies } from '@/lib/auth-cookies';
import { createClient } from '@supabase/supabase-js';

export async function POST() {
  try {
    // 1. 创建 Supabase admin client（使用 service role key）
    const supabase = createClient(
      process.env.NEXT_PUBLIC_SUPABASE_URL!,
      process.env.SUPABASE_SERVICE_ROLE_KEY!,
      {
        auth: {
          autoRefreshToken: false,
          persistSession: false,
        },
      }
    );

    // 2. 尝试从 cookie 中获取 access token
    const cookieHeader = request.headers.get('cookie') || '';
    const tokenMatch = cookieHeader.match(/sb-access-token=([^;]+)/);
    const accessToken = tokenMatch ? tokenMatch[1] : null;

    // 3. 如果存在 access token，调用 Supabase 服务端 signOut
    //    注意：使用 service role key 调用 signOut 会撤销该用户的所有 session
    if (accessToken) {
      // 先获取用户，然后撤销其 session
      const { data: { user }, error: userError } = await supabase.auth.getUser(accessToken);
      if (user && !userError) {
        // 使用 admin API 撤销用户的所有 session
        await supabase.auth.admin.signOut(user.id);
      }
    }

    // 4. 清除所有相关 cookie（包括可能存在的其他 Supabase cookie）
    const response = NextResponse.json({ success: true, message: '已退出登录' });
    clearAuthCookies(response);
    
    // 额外清除可能存在的其他 Supabase cookie
    response.cookies.set('sb-provider-token', '', { 
      httpOnly: true, secure: process.env.NODE_ENV === 'production', 
      sameSite: 'lax', path: '/', maxAge: 0 
    });
    response.cookies.set('sb-auth-token', '', { 
      httpOnly: true, secure: process.env.NODE_ENV === 'production', 
      sameSite: 'lax', path: '/', maxAge: 0 
    });

    return response;
  } catch (error) {
    console.error('Logout error:', error);
    return NextResponse.json({ error: '退出失败' }, { status: 500 });
  }
}
```

### src/app/api/auth/me/route.ts（优化版本）

```typescript
import { NextResponse } from 'next/server';
import { createClient } from '@supabase/supabase-js';

export const dynamic = 'force-dynamic';

export async function GET(request: Request) {
  try {
    const cookieHeader = request.headers.get('cookie') || '';
    const tokenMatch = cookieHeader.match(/sb-access-token=([^;]+)/);
    const token = tokenMatch ? tokenMatch[1] : null;

    if (!token) {
      return NextResponse.json({ error: '请先登录' }, { status: 401 });
    }

    const supabase = createClient(
      process.env.NEXT_PUBLIC_SUPABASE_URL!,
      process.env.SUPABASE_SERVICE_ROLE_KEY!,
      {
        auth: {
          autoRefreshToken: false,
          persistSession: false,
        },
      }
    );

    // 使用 getUser 验证 token 有效性
    // 注意：getUser 会验证 token 签名和过期时间，但不会检查 session 是否被撤销
    // 为了更严格的验证，可以额外检查 session 状态
    const { data: { user }, error } = await supabase.auth.getUser(token);

    if (error || !user) {
      return NextResponse.json({ error: '认证失败' }, { status: 401 });
    }

    // 额外验证：检查该用户是否有活跃 session
    // 使用 admin API 获取用户的 session 列表
    const { data: sessions, error: sessionsError } = await supabase.auth.admin.listSessions();
    
    // 如果无法获取 sessions（权限问题），至少确保 token 本身有效
    if (!sessionsError && sessions) {
      const hasActiveSession = sessions.some(s => 
        s.user_id === user.id && 
        s.access_token === token &&
        new Date(s.expires_at) > new Date()
      );
      
      if (!hasActiveSession) {
        return NextResponse.json({ error: '会话已过期' }, { status: 401 });
      }
    }

    return NextResponse.json({
      success: true,
      user: {
        id: user.id,
        email: user.email,
        phone: user.user_metadata?.phone || user.phone || null,
        nickname: user.user_metadata?.nickname || '用户' + (user.email?.split('@')[0]?.slice(-4) || '')
      }
    });
  } catch (err) {
    console.error('[auth/me] Error:', err);
    return NextResponse.json({ error: '服务器错误' }, { status: 500 });
  }
}
```

### src/lib/auth-cookies.ts（优化版本）

```typescript
import { NextResponse } from 'next/server';

const COOKIE_OPTIONS = {
  httpOnly: true,
  secure: process.env.NODE_ENV === 'production',
  sameSite: 'lax' as const,
  path: '/',
};

export function setAuthCookies(
  response: NextResponse,
  accessToken: string,
  refreshToken: string,
  expiresAt: number
): void {
  const now = Math.floor(Date.now() / 1000);
  const expiresIn = Math.max(expiresAt - now, 3600);
  const maxAge = 30 * 24 * 60 * 60;

  response.cookies.set('sb-access-token', accessToken, {
    ...COOKIE_OPTIONS,
    maxAge: expiresIn,
  });
  response.cookies.set('sb-refresh-token', refreshToken, {
    ...COOKIE_OPTIONS,
    maxAge: maxAge,
  });
}

export function clearAuthCookies(response: NextResponse): void {
  // 使用与设置时完全相同的选项来清除 cookie
  response.cookies.set('sb-access-token', '', {
    ...COOKIE_OPTIONS,
    maxAge: 0,
  });
  response.cookies.set('sb-refresh-token', '', {
    ...COOKIE_OPTIONS,
    maxAge: 0,
  });
}

export function parseAccessTokenFromCookie(headers: Headers): string | null {
  const cookieHeader = headers.get('cookie') || '';
  const match = cookieHeader.match(/sb-access-token=([^;]+)/);
  return match ? match[1] : null;
}
```

# 关键说明

1. **`supabase.auth.admin.signOut(user.id)`**：使用 service role key 调用此方法会撤销该用户的所有活跃 session，确保即使旧的 access token 被泄露也无法使用
2. **Cookie 清除一致性**：确保清除 cookie 时使用与设置时完全相同的选项（httpOnly, secure, sameSite, path）
3. **额外 cookie 清理**：Supabase 可能设置其他 cookie（如 `sb-provider-token`），一并清除
4. **me 路由增强**：添加了 session 验证逻辑，确保 token 对应的 session 仍然有效

这个方案最小化改动，只修改了 logout 和 me 路由，不影响 login/register 逻辑。
