## 任务 1（P1）：中文搜索 pg_trgm 全文索引，修 `/api/jobs?keyword=` 15s 超时

### 背景
- 日程测试报告 `职途星/测试报告/2026-05-27-测试报告.md` 显示：`/api/jobs?keyword=中文关键词` 耗时 **15s 超时**
- 已确认（grep 验证）当前 `src/app/api/jobs/route.ts` 第 290-298 行用 `ilike('字段', '%keyword%')` 前后通配，**普通 btree 索引无效**，必须用 `pg_trgm` GIN 索引才能加速
- 当前 keyword 触发 6 个并发查询合并去重，**实际命中的字段**（已 grep 确认，别想当然）：
  - `job_title`（ilike `%keyword%`）
  - `industry`（ilike `%keyword%`）
  - `hard_skills`（contains 数组 + ilike JSON 字符串 fallback）
  - `soft_skills`（同上）

### 目标
让 `/api/jobs?keyword=Python` 这种中英文搜索从 15s 降到 **< 1.5s**。

### 步骤

**Step 1：先 grep 二次确认字段名**（防字段名幻觉）
```bash
grep -nE "(ilike|contains)" src/app/api/jobs/route.ts | head -20
```
应该看到 `job_title` / `industry` / `hard_skills` / `soft_skills` 四个字段。**如果实际字段名跟这里写的不一样，以代码为准，立刻停下来在 commit message 里说明**。

**Step 2：新建 SQL 迁移文件**
- 路径：`sql/migrations/2026-05-27_jd_pg_trgm.sql`
- 内容要求：
  - 开头注释：执行时间、用途、风险等级（低，CREATE INDEX 不锁表）
  - **必须先**：`CREATE EXTENSION IF NOT EXISTS pg_trgm;`
  - 给 `job_title` 加 GIN trgm 索引：
    ```sql
    CREATE INDEX IF NOT EXISTS idx_jd_job_title_trgm 
    ON public.job_descriptions USING gin (job_title gin_trgm_ops);
    ```
  - 给 `industry` 加 GIN trgm 索引（同上模式，索引名 `idx_jd_industry_trgm`）
  - `hard_skills` / `soft_skills` 是 JSON/数组字段，**不要加 trgm**（trgm 是文本索引，对 JSON 强行加会出错）；如果它们是 jsonb 类型可以加 GIN 默认索引，**但要先确认类型**：
    ```sql
    SELECT column_name, data_type 
    FROM information_schema.columns 
    WHERE table_name = 'job_descriptions' 
      AND column_name IN ('hard_skills', 'soft_skills');
    ```
    把这条查询作为注释**写在 SQL 文件里**让主人手动跑一下决定（先不加 hard_skills/soft_skills 索引，等主人确认类型）
  - 包 `BEGIN; ... COMMIT;`
  - 末尾加校验：
    ```sql
    SELECT indexname, indexdef 
    FROM pg_indexes 
    WHERE tablename = 'job_descriptions' 
      AND indexname LIKE 'idx_jd_%_trgm';
    -- 期望返回 2 行
    ```
  - 顶部注释明确写：`-- 执行者：人工在 Supabase Dashboard SQL Editor 执行（不要 Trae 自动跑）`

**Step 3：代码端无需改动**
- 现有 `ilike('job_title', '%${keyword}%')` 加完索引后会**自动走 GIN trgm**，PostgreSQL 查询规划器自己识别
- **不要**改 `src/app/api/jobs/route.ts` 的查询逻辑（除非你看到明显 bug）

**Step 4：验证 build**
```bash
pnpm run build
```
EXIT=0 才能继续。

**Step 5：commit + push**
- commit message：`perf(jd): 新增 pg_trgm GIN 索引（job_title+industry），修复中文搜索 15s 超时`
- push 到 `main`

- `CREATE EXTENSION pg_trgm` 需要 Supabase 项目允许（绝大多数 Supabase 项目默认允许）
