# Trae 工单 007 - B4：company 字段空值前端兜底显示

**日期**：2026-05-27  
**预计耗时**：30-45 分钟  
**优先级**：P1（与 B3 并行，文件不冲突，可同时跑）  
**前置依赖**：无

---

## 背景

5/27 晚把 Coze 旧库 19997 条 hrjdkey 数据入库 supabase 后，真实 JD 总量从 659 涨到 20656，6/5 答辩硬指标 ≥20000 已达成。

但 hrjdkey 原始数据**没有公司名字段**（Coze 老表 20 个字段就缺这一项，raw_jd 只存职责+要求，整段文本不出现"公司"二字）。抽 50 条让 DeepSeek 试反推也无解——无米之炊。

**决策**：保持 `company` 字段为 `NULL`，前端遇到 NULL 时**用 `行业 · 公司性质` 作为兜底文案**，不显示"未知公司"/"Unknown"/空白行。

---

## 数据库现状（背景资料，无需改）

`job_descriptions` 表里：
- `company`：~20000 行为 NULL（hrjdkey 那批），659 行有真公司名（老 gxrc 那批）
- `industry`：**100% 填充**（hrjdkey 20+ 行业全有，gxrc 也有）
- `company_type`：hrjdkey **100% 填充**（如"民营企业 / 外资企业 / 国企央企 / 股份制企业"），gxrc 部分有

所以兜底逻辑**安全可用**。

## 改动清单

### 任务 1：UI 组件兜底 - `src/components/cards/JdMatchCard.tsx`

**当前**（约第 101 行）：
```tsx
{companyName && <div className="text-sm text-gray-600 mt-0.5">{companyName}</div>}
```

**改为**：
```tsx
{(() => {
  const fallback = [data.industry, data.company_type].filter(Boolean).join(' · ');
  const display = companyName || fallback;
  return display ? <div className="text-sm text-gray-600 mt-0.5">{display}</div> : null;
})()}
```

同步在 `JdMatchData` interface（约第 9-20 行）加字段：
```ts
company_type?: string;
```

### 任务 2：UI 组件兜底 - `src/app/profile/favorites/page.tsx`

**当前**（约第 11 行的 interface，140-141 行的渲染）：
- interface 里只有 `company: string | null`
- 渲染：`{favorite.company && (<p className="text-gray-600 mb-2">{favorite.company}</p>)}`

**改为**：
- interface 补：`industry?: string | null; company_type?: string | null;`
- 渲染：
```tsx
{(() => {
  const fallback = [favorite.industry, favorite.company_type].filter(Boolean).join(' · ');
  const display = favorite.company || fallback;
  return display ? <p className="text-gray-600 mb-2">{display}</p> : null;
})()}
```

### 任务 3：UI 组件兜底 - `src/app/profile/page.tsx`

约第 813 行：
```tsx
{favorite.company && <p className="text-sm text-gray-600">{favorite.company}</p>}
```

同上改造（注意区分这个文件里第 341/346 行的 `item.company` 是**实习经历**的 company，不要动；只改第 813 行收藏卡片那处）。

收藏 item 的 TS 类型同样补 `industry?: string | null; company_type?: string | null;`。

### 任务 4：API 兜底 - `src/app/api/search-jd/route.ts`

**当前**（约第 75 行 select、92 行 transform）：
```ts
.select('job_title, company, city, salary_range, industry, responsibilities, fresh_graduate_friendly, source_platform')
// ...
company_name: (job.company as string) || 'Unknown',
```

**改为**：
- select 补 `company_type`
- transform：
```ts
const fallback = [job.industry, job.company_type].filter(Boolean).join(' · ');
company_name: (job.company as string) || fallback || '行业未填',
company_type: (job.company_type as string) || '',
```

把硬编码 `'Unknown'` 干掉。

### 任务 5：API 兜底 - `src/app/api/match/route.ts`

约第 69 行 + 第 249 行两处 `.select(...)`：在字段列表里**补 `company_type`**。

返回结构里（约 174 行、277 行）：把 `company: jd.company` 改为：
```ts
company: jd.company || [jd.industry, jd.company_type].filter(Boolean).join(' · ') || null,
```

这样前端 `JdMatchCard` 即使没改也能立刻显示兜底文案（双保险）。

### 任务 6：收藏 API 字段返回 - `src/app/api/favorites/route.ts`

当前 `.select('*')` 已经包含 `industry / company_type`，无需改。但 `[id]/route.ts` 如有 select 显式列字段的，要补 `industry, company_type`。Trae 自行检查。
