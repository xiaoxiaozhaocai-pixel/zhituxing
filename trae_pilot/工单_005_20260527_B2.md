## 任务 3（P0）：新建 `src/app/api/cron/jd-sync/route.ts`

### 文件
`src/app/api/cron/jd-sync/route.ts`

### 内容
```ts
import { NextRequest, NextResponse } from 'next/server';
import { syncOfficialJobs } from '@/lib/jd-sync';

export const runtime = 'nodejs'; // 必须 Node runtime，不用 Edge

export async function POST(request: NextRequest) {
  // 1. Secret 校验（防爬）
  const cronSecret = process.env.CRON_SECRET;
  if (!cronSecret) {
    return NextResponse.json({ code: 500, message: 'CRON_SECRET 未配置' }, { status: 500 });
  }
  const authHeader = request.headers.get('authorization');
  if (authHeader !== `Bearer ${cronSecret}`) {
    return NextResponse.json({ code: 401, message: 'unauthorized' }, { status: 401 });
  }

  // 2. 解析 query 参数
  const { searchParams } = new URL(request.url);
  const limit = Math.min(500, parseInt(searchParams.get('limit') || '100'));
  const dryRun = searchParams.get('dryRun') === 'true';

  try {
    const result = await syncOfficialJobs({ limit, dryRun });
    return NextResponse.json({ code: 200, data: result });
  } catch (err) {
    console.error('cron jd-sync error:', err);
    return NextResponse.json({ code: 500, message: 'cron 执行失败', error: String(err) }, { status: 500 });
  }
}

// GET 走和 POST 一样的逻辑（方便外部 cron 服务用 GET 触发）
export async function GET(request: NextRequest) {
  return POST(request);
}
```

### 要求
- **必须 `runtime = 'nodejs'`**（采集会用 fetch 长时间执行，Edge 会超时）
- **必须有 Secret 校验**（防止任何人调你的 API 让你帮他爬数据）
- 支持 GET 和 POST（GET 方便外部 cron 服务调）
- 默认 `limit=100, dryRun=false`，上限 500

## 任务 4（P1）：写 README 说明 cron 触发器配置选项

### 文件
新建 `docs/cron-jd-sync-setup.md`

### 内容要求
解释 3 种触发器方案的优劣 + 配置步骤（**不强制配置任何一个**，让主人后续自己选）：

1. **方案 A：cron-job.org（免费推荐）**
   - 注册 → 新建 cron → URL: `https://zhituxing.zeabur.app/api/cron/jd-sync?limit=100`
   - HTTP 方法 GET，Headers 加 `Authorization: Bearer <CRON_SECRET>`
   - 频率：每天 03:00（错峰）
   - 优点：完全免费，配置 1 分钟
   - 缺点：依赖第三方稳定性

2. **方案 B：Zeabur Cron Service**
   - 在 Zeabur 控制台新建一个 cron service 类型的服务
   - 指向同一个仓库，启动命令：`curl -X POST -H "Authorization: Bearer $CRON_SECRET" https://zhituxing.zeabur.app/api/cron/jd-sync`
   - 优点：跟主服务一个平台好管理
   - 缺点：占 Zeabur 配额，可能要钱

3. **方案 C：本机 crontab（兜底）**
   - 用户自己电脑加 cron：`0 3 * * * curl -X POST -H "Authorization: Bearer xxx" https://zhituxing.zeabur.app/api/cron/jd-sync`
   - 优点：完全免费、可控
   - 缺点：电脑必须开机

### 必须额外写
- **如何生成 CRON_SECRET**：`openssl rand -hex 32` 复制到 Zeabur 环境变量
- **如何在 Zeabur 加环境变量**：控制台 → service → Environment Variables → 新增 `CRON_SECRET`
- **如何手动测试**（不等 cron）：本地 curl 命令示例（dryRun=true 安全）
