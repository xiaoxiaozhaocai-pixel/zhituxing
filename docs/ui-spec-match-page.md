# /match 页面交互规范

> 维护者：架构组
> 最后更新：2026-05-28（commit 4a47399）
> 历史背景：BUG-3 复核（2026-05-28）发现三方对卡片交互有三套不同预期，统一后形成本规范。

## 1. 页面定位

`/match` —— **岗位智能匹配结果页**。基于用户技能向量与 JD 关键技能计算匹配度，按分数倒序展示推荐岗位卡片，**不包含独立详情页路由**（无 `/jobs/[id]`）。

## 2. 交互模型：Accordion 内联展开

> ⚠️ 本页**不使用** Dialog 模态框，也**不跳转**到独立详情页。
> 所有岗位详情均通过 **accordion 内联展开** 在卡片内呈现。

### 2.1 收起态（默认）

- 卡片高度约 **152px**
- 显示：匹配度环 / 岗位标题 / 城市 · 行业 · 薪资估算 / 技能标签摘要（最多 3+2 个）/ 右上"详情"按钮
- 右侧"详情"按钮显示 `ChevronDown` 图标 + 文字"详情"

### 2.2 展开态

- 卡片高度约 **441px**
- 收起态全部内容 +
  - 「已匹配技能」全量列表
  - 「技能缺口」全量列表
  - 「薪资估算」详细面板（基于匹配度的最低/最高/中位数）
- 右侧"详情"按钮显示 `ChevronUp` 图标 + 文字"详情"

### 2.3 触发方式（三种等价）

| 触发方式 | 行为 |
| --- | --- |
| **整张卡片点击**（任意空白区域、标题、技能标签、匹配度环） | toggle 展开/收起 |
| **"详情"按钮点击** | toggle 展开/收起（与卡片点击同步，不重复触发） |
| **键盘 Enter / Space**（卡片获得焦点时） | toggle 展开/收起 |

### 2.4 状态管理

- 同一时刻**只允许一张卡片展开**（state: `expandedId: number \| null`）
- 点击新卡片时，前一张自动收起
- 再次点击同一卡片，收起

## 3. 无障碍（a11y）

| 属性 | 值 | 说明 |
| --- | --- | --- |
| `role` | `button` | Card 标识为可操作元素 |
| `tabIndex` | `0` | 卡片本体可 tab 聚焦 |
| `aria-expanded` | `isExpanded`（boolean） | 屏幕阅读器播报展开/收起状态 |
| `aria-label` | `"{岗位名}，匹配度{N}%，点击{展开/收起}详情"` | 完整语义描述 |
| `focus-visible:ring-2` | Tailwind | 键盘焦点指示器（鼠标点击不显示） |
| "详情"按钮 `aria-hidden` | `true` | 避免与卡片本体重复出现在 tab 顺序 |
| "详情"按钮 `tabIndex` | `-1` | 同上 |

## 4. 验收要点（给 reviewer / Browser Use）

测试 /match 页面时，请按本规范预期，**不要**期望以下行为：

- ❌ 点击卡片跳转到 `/jobs/[id]` → 不存在该路由
- ❌ 点击卡片弹出 Dialog 模态框 → 已确认废弃此方案
- ❌ 卡片在 hover 时显示"查看详情"提示链接 → 改用整卡 cursor-pointer + 阴影反馈

正确验收清单：

- ✅ 卡片 hover 应有阴影加深 + 边框颜色变深（`hover:shadow-lg hover:border-orange-300`）
- ✅ 卡片 cursor 应为 `pointer`
- ✅ 点击卡片任意位置（除"详情"按钮内）应展开/收起
- ✅ 点击"详情"按钮应有相同效果（不冒泡导致双触发）
- ✅ Tab 键应能聚焦到卡片，Enter/Space 应触发展开
- ✅ 同一时刻只有一张卡片展开
- ✅ Console 应 0 错误
