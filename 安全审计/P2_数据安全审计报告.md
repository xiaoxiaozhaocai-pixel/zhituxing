### [P2-CRIT-1] CSRF 防护函数已实现但未全局启用

**严重程度**：严重（上线前必须修）  
**漏洞类型**：CSRF（跨站请求伪造）  
**影响范围**：所有 POST/PUT/DELETE/PATCH API 路由  
**文件位置**：
- `src/lib/csrf.ts`（防护函数已就绪）
- `src/middleware.ts`（**不存在**，无法全局拦截）

**攻击场景**：攻击者在恶意网站嵌入自动提交的表单，诱导已登录用户访问。由于缺少全局 CSRF 中间件，攻击者可利用用户的登录态执行订单创建、个人信息修改、简历优化等状态变更操作。虽然已登录用户需携带 Cookie（SameSite=Lax），但在同站攻击场景下仍然脆弱。

**当前状态**：`csrf.ts` 已实现完整的 Origin/Referer 验证函数，但没有在任何地方被调用。项目**不存在 `middleware.ts` 文件**，无法在全局层面拦截恶意请求。

**修复代码**：创建 `src/middleware.ts`：
```typescript
import { NextRequest, NextResponse } from 'next/server';
import { validateCSRF } from '@/lib/csrf';

export function middleware(request: NextRequest) {
  // 对状态变更请求执行 CSRF 检查
  const csrfResult = validateCSRF(request);
  if (csrfResult) {
    return csrfResult; // 返回 403
  }

  // 安全响应头
  const response = NextResponse.next();
  response.headers.set('X-Frame-Options', 'DENY');
  response.headers.set('X-Content-Type-Options', 'nosniff');
  response.headers.set('Referrer-Policy', 'strict-origin-when-cross-origin');

  return response;
}

export const config = {
  matcher: [
    // 匹配所有 API 路由
    '/api/:path*',
    // 匹配管理后台
    '/admin/:path*',
  ],
};
```

**修复验证**：
```bash
# 模拟跨站 POST 请求（应返回 403）
curl -X POST https://zhituxing.tech/api/orders \
  -H "Content-Type: application/json" \
  -H "Origin: https://evil.example.com" \
  -d '{"plan":"monthly","payment_method":"wechat"}'
# 预期：HTTP 403 + '跨站请求被拒绝'
```

### [P2-MED-1] AIResponseRenderer 组件的 dangerouslySetInnerHTML 使用需审计

**严重程度**：中等（尽快修复）  
**漏洞类型**：XSS  
**影响范围**：所有 AI 聊天回复的渲染  
**文件位置**：`src/components/AIResponseRenderer.tsx`

**风险描述**：AI 聊天回复通过 `AIResponseRenderer` 渲染，如果使用了 `dangerouslySetInnerHTML`，即使内容经过 AI 端过滤，仍存在 XSS 风险——恶意 prompt 可能诱导 AI 输出恶意 HTML。

**验证需求**：需确认 `AIResponseRenderer.tsx` 中的 HTML 渲染是否经过 `sanitizeHtml()` 处理。建议添加防护：

```typescript
// 确保所有 AI 输出经过净化
import { sanitizeHtml } from '@/lib/sanitize';

function renderAIResponse(content: string) {
  const cleanHtml = sanitizeHtml(content);
  return <div dangerouslySetInnerHTML={{ __html: cleanHtml }} />;
}
```

### [P2-MED-3] 敏感操作缺少 CSRF Token（double-submit cookie 模式）

**严重程度**：中等（尽快修复）  
**漏洞类型**：CSRF  
**影响范围**：订单提交、简历优化等关键操作  
**文件位置**：多个 API 路由

**风险描述**：`csrf.ts` 仅检查 Origin/Referer 头，未实现 double-submit cookie 模式。虽然 Origin 验证对大多数浏览器有效，但某些老旧浏览器或特殊配置可能不发送 Origin 头。对于关键操作（支付/订单），建议增加 CSRF Token 验证。

**建议**：在 `csrf.ts` 中增加 double-submit cookie 验证作为第二层防护：
```typescript
export function validateCSRFToken(request: NextRequest): boolean {
  const cookieToken = request.cookies.get('csrf-token')?.value;
  const headerToken = request.headers.get('x-csrf-token');
  if (!cookieToken || !headerToken) return false;
  return cookieToken === headerToken;
}
```

### [P2-MED-4] 错误信息泄露 - 数据库错误详情可能暴露给前端

**严重程度**：中等（尽快修复）  
**漏洞类型**：信息泄露  
**影响范围**：部分 API 路由  
**文件位置**：
- `src/app/api/user/profile/route.ts`（返回 `error.details`）
- `src/app/api/admin/dashboard/ai-insights/route.ts`（返回 `String(error)`）
- `src/app/api/orders/route.ts`（部分错误信息）

**风险描述**：某些 API 错误响应中包含数据库错误详情，可能暴露内部结构信息。

**当前代码**（`user/profile/route.ts`）：
```typescript
return NextResponse.json({ 
  code: 500, 
  error: error.message || '保存失败',
  details: error  // ❌ 暴露内部错误详情
}, { status: 500 });
```

**修复代码**：
```typescript
// 统一错误响应
return NextResponse.json({
  code: 500,
  message: '服务器内部错误，请稍后重试'
}, { status: 500 });

// 详细错误仅写入日志
console.error('[user/profile] 保存失败:', error);
```

### [P2-MIN-2] 文件上传缺乏内容验证（MIME 类型绕过）

**严重程度**：轻微（后续迭代）  
**文件位置**：`src/app/api/orders/upload/route.ts`

**当前状态**：文件上传仅检查 MIME type（`file.type`），未验证文件魔数（magic bytes）。攻击者可能伪造 Content-Type 上传恶意文件。

**建议**：增加魔数验证：
```typescript
const MAGIC_BYTES: Record<string, number[]> = {
  'jpg': [0xFF, 0xD8, 0xFF],
  'png': [0x89, 0x50, 0x4E, 0x47],
  'webp': [0x52, 0x49, 0x46, 0x46],
};

function validateMagicBytes(bytes: Uint8Array, ext: string): boolean {
  const magic = MAGIC_BYTES[ext];
  if (!magic) return false;
  return magic.every((byte, i) => bytes[i] === byte);
}
```
